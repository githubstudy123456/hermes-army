## 二、各部门待办任务

### 市场部（CMO）— 负责市场调研和客户验证

- [ ] **M1** 验证 ContractAI 目标客户：联系 5 家创业公司，询问合同审查需求和价格接受度
- [ ] **M2** 验证 ContentStudio 目标客户：在小红书发一篇测试内容，收集咨询量
- [ ] **M3** 竞争对手定价调研：调研秘塔、幂律、天眼查的合同审查定价，**补充功能覆盖范围说明**
- [ ] **M4** 销售渠道设计：哪些渠道最适合上述 7 款产品
- [ ] **M5** 目标客户画像确认：7 款产品各选 3 个真实客户案例
- [ ] **M6** ContentStudio发布能力边界确认：是否能实际登录客户账号代发小红书/知乎/公众号，**提交操作流程说明**

### 研究部（CIO）— 负责技术可行性和数据源

- [ ] **R1** ContractAI 合同审查能力验证：测试 5 份真实合同，输出质量报告
- [ ] **R2** DueDiligence 数据源确认：天眼查 API 成本、裁判文书网数据完整性
- [ ] **R3** EquityWatch 研究质量验证：对比我们输出的报告 vs 券商研报
- [ ] **R4** PolicyGuard 关键词配置：各行业政策监测关键词库建立
- [ ] **R5** 交付质量标准：每个产品制定**可测量**的交付标准

### 信息收集组（Collector）— 负责行业监测

- [ ] **C1** 重点行业筛选：哪 3 个行业客户最有可能买单（金融、医疗、消费）
- [ ] **C2** K-pop 资讯 cron 验证：现有 cron 是否可以直接作为产品交付
- [ ] **C3** 政策数据源清单：政府网站列表 + 更新频率 + 数据完整性

### 法务部（CLO）— 负责合同模板和合规

- [ ] **L1** 客户合同模板：标准服务协议（明确交付物、免责条款）
- [ ] **L2** 免责声明模板：各产品通用免责声明，**与产品文档保持一致**
- [ ] **L3** 隐私政策：客户数据处理规则
- [ ] **L4** 定价合规性确认：现有定价是否需要资质

### 审核部（CRO）— 负责质量审计（严格模式）

- [ ] **A1** 文档审核：审核以上所有文档的完整性和准确性
- [ ] **A2** 定价审核：审核报价是否合理、有竞争力
- [ ] **A3** 风险审核：审核各产品风险提示是否充分
- [ ] **A4** 质量红线检查：所有提交文档必须通过以下检查项
  - [ ] 日期是否为当前年份（2026）
  - [ ] 交付边界是否清晰（谁做、做什么、不做什么）
  - [ ] 成本估算是否与AI定位一致（避免写"人工X小时"）
  - [ ] 风险提示是否有操作指引（不只是"仅供参考"）
