# 中国改革开放报告生成

## 触发条件
用户要求生成改革开放以来（中国1978年至今）每年重点事件、制度变迁、改革里程碑的编年报告时使用本技能。

## 最终版格式（v5 — 用户确认版）

经过4轮迭代，用户最终确认的格式：

```
年份 | 事件标题 | 关键说明（1-2句）
```

**核心原则：**
- 每年重要事件不限制条数，有几个重要事件就写几条
- 制度变化在当年事件描述中自然体现，**不单独写制度条目**
- 不写具体法律名称（不用书名号《》），用制度层面的描述
- 示例：写"股权分置改革试点启动，解决A股全流通问题"而非"《证券法》颁布"

**正确理解用户意图的三个层次：**
1. ❌ "每年一条"（误解，用户说"重要的事件都可以写"）
2. ❌ 制度专篇/分类列表（用户说"不是这种具体的某一个内容"，不要单独制度条目）
3. ✅ 自然编年史 + 制度变化在事件中体现 = **v5格式**

## 技能使用说明

### 调用方式
直接生成报告内容，不需要额外 delegate_task — skill 内置完整编年数据，直接读取 `/home/ubuntu/china_reform_40years_1978_2025_v5.txt`（113条记录，1978-2025）输出即可。

### delegate_task 使用场景（仅用于需要扩展数据时）
如用户要求延伸至最新年份（2026），再通过 delegate_task 并行查询 web 数据后合并，不要在有现成数据时重复劳动。

### 格式选择决策树
1. 用户说"每年重点发生了啥" → 直接上 v5 纯文本格式，不用问
2. 用户说"给我出份报告" → 同上，输出纯文本
3. 用户指定格式（如表格/PPT） → 按指定格式
4. 两者都说"改革开放的激荡40年" → v5格式，标题可加"激荡"修饰词
