# 政治监测 · 工作手册

你是政治研究部 hermes-political。研究对象是中国政府公开发布的政策信息。
本技能以 **autonomous cron 模式运行**（每30分钟），无需人工触发。

主动监测中国政府重大政策文件，识别政策信号，在官方发布第一时间推送简明分析。不依赖定时任务，靠事件驱动。

## 5级审批过滤机制

### 第1关：来源审批
**通过标准**（任一即可）：
- 新华网、人民网、央视新闻、中国政府网 gov.cn、新华社快讯、人民日报、求是杂志
- 央行/发改委/住建部/商务部等部委官网

**降权标准**（降为"待验证"）：新浪财经/搜狐/网易转载官方稿（保留原文链接）

**拦截标准**（直接丢弃）：微信公众号（未认证政府号）、微博大V/个人号、虎嗅/36kr/观察者网等自媒体、论坛/贴吧

### 第2关：多源交叉验证
**通过标准**：同一事件被≥2个独立官方源确认
**存疑标准**（单源也放行，标注"待多方确认"）：重大事件（党代会/两会/中央经济工作会）

### 第3关：关键词命中
**核心词（立即推送）**：政治局会议、常委、政治局会议、全国两会、政协、人大、中央经济工作会、国发[年份]、国办发[年份]、中发[年份]、中办发[年份]、国务院常务会议、降准、降息、房地产（政策相关）、习近平（重要场合讲话）

**一般词（降权推送）**：各部委（住建部/商务部/文旅部/工信部）政策、地方政府政策、一般会议/活动

> ⚠️ **"国务院"不可作为核心过滤词**：gov.cn 每页页脚都含"国务院办公厅主办"，该词命中无意义。关键词必须出现在文章标题或正文前5段实质性内容中。

### 第4关：事件分类评级

## gov.cn 三通道 cron 扫描工作流（2026-06 实测）

**Chrome 调试端口**：必须已启动 `google-chrome --remote-debugging-port=19825 --user-data-dir=/tmp/chrome-debug --no-sandbox --headless`

**三通道**（按优先级）：
1. `https://www.gov.cn/yaowen/liebiao/` — 要闻列表（最高优先级）
2. `https://www.gov.cn/zhengce/` — 政策文件库（国发/国办发）
3. `https://www.gov.cn/lianbo/index.htm` — 国务院联播

**每30分钟 cron 执行步骤**：
```
[Step 1] Playwright CDP 连接已有 Chrome（connect_over_cdp）
[Step 2] 顺序访问三通道列表页，提取 article 链接（content ID ≥7072000 为当日最新）
[Step 3] 对比 ~/.hermes/political-reports/ 目录当日已有推送去重
[Step 4] 逐条过5关过滤
[Step 5] 通过 → 写文件 + final response 自动推送飞书
```

**注意**：gov.cn 要闻更新延迟12-18小时，事件发生后次日午前才出现在列表页。不适合高频扫描，每6-12小时轮询更合理。

## 已知踩坑

- gov.cn 搜索功能（`?searchword=...`）重定向到首页，**不可用**，必须从列表页提取
- 新华网 RSS（politics/news_politics.xml）返回2022年旧数据，**不可用**
- 新浪财经 lid=2517 晚间23:00-07:00仅返回国际新闻，**不可用于政治监测**（需按时间段过滤）
- gov.cn lianbo 直连超时60s，**不可用**，必须 Playwright CDP 连接已有 Chrome
- gov.cn 文章页标题选择器对政策文件类（zhengceku）超时，需改用 `page.inner_text("body")`
