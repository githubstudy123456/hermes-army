## 標記章節 A：訴訟日程監控 (Docket Watcher)

*來源：docket-watcher*

### 中國訴訟時效規定（民法典）
| 時效類型 | 期限 | 起算點 |
|---------|------|--------|
| 普通訴訟時效 | 3年 | 權利人知道或應當知道權利受損及義務人之日 |
| 特別訴訟時效 | 1年/2年等 | 法律規定或當事人約定 |
| 最長訴訟時效 | 20年（特殊可延長） | 權利受損之日 |
| 仲裁時效 | 3年（勞動仲裁1年） | 同普通時效 |

### 民事訴訟關鍵節點
| 階段 | 期限要求 | 備註 |
|------|---------|------|
| 立案 | 7日內決定是否立案 | 民訴法第123條 |
| 答辯期 | 15日內提交答辯狀 | 民訴法第125條 |
| 舉證期 | 不少於15日（簡易程序可縮短） | 民訴法第68條 |
| 一審判決上訴期 | 15日內 | 民訴法第170條 |
| 生效判決申請執行 | 2年內 | 民訴法第239條 |
| 保全錯誤的補救 | 30日內 | 民訴法第108條 |

### 開庭前提醒邏輯
```
開庭前7天 → 首次提醒（準備材料清單）
開庭前3天 → 第二次提醒（出庭人員確認）
開庭前1天 → 最終確認（證據/代理詞準備情況）
開庭當天 → 當日提醒（時間地點再次確認）
判決送達 → 即時提醒（上訴期限開始計算）
```

### 操作步驟
1. 導入案件清單（案件管理系統數據或對話輸入）
2. 計算關鍵日期（開庭日期、舉證期限截止日、答辯期截止日、上訴期限、訴訟時效屆滿日）
3. 生成日程監控表（按日期排序，標注緊迫程度）
4. 推送提醒（提前7天/3天/1天分階段）
5. 訴訟時效追蹤（標記即將屆滿的訴訟時效）

### Pitfalls
1. **時效計算錯誤**：普通時效3年自知道或應當知道起算，需核實起算點
2. **舉證期限遺漏**：逾期舉證可能被駁回或承擔不利後果
3. **公告送達**：被告下落不明時公告期60日，需單獨計算
4. **上訴期限**：判決書送達之日起15日內上訴，錯過喪失上訴權
5. **保全期限**：訴前保全30日內不起訴則解除

## 標記章節 B：訴訟文書起草 (Pleading Draft)

*來源：pleading-draft*

### 訴狀基本要素（民事訴訟法第122條）
- 原告姓名、住所、聯繫方式
- 被告姓名、住所、聯繫方式
- 訴訟請求和事實理由
- 證據清單

### 答辯狀基本要素
- 對原告訴訴訟請求的承認或否認
- 所依據的事實和證據
- 答辯理由和法律依據
- 反訴請求（如有）

### 常見訴訟類型要點
| 案件類型 | 管轄規定 | 舉證要點 |
|---------|---------|---------|
| 合同糾紛 | 被告住所地/合同履行地 | 合同文本、履行憑證、違約證據 |
| 侵權糾紛 | 侵權行為地/被告住所地 | 侵權事實、損失計算、责任依據 |
| 物權糾紛 | 不動產所在地 | 權屬證明、占有人證明 |
| 婚姻糾紛 | 被告住所地/婚姻登記地 | 感情破裂證據、財產清單 |

### 操作步驟
1. 收集案件材料（合同、往來函件、證據文件）
2. 分析案件事實（梳理時間線、識別法律關係、確定訴訟請求）
3. 起草訴訟文書（選擇文書模板、填寫當事人信息、撰寫訴訟請求）
4. 法律依據論證（引用相關法律條文、援引類似判例）
5. 生成文書草稿（輸出結構完整的訴訟文書，標注需律師審核的重點）

### Pitfalls
1. **管轄錯誤**：向無管轄權法院起訴會被駁回，需確認管轄連接點
2. **訴訟請求不明確**：請求不明確會被要求補正，影響立案
3. **訴訟時效**：確認未超過訴訟時效，必要時先發催告函
4. **證據目錄**：證據需在舉證期內提交，逾期可能不被採納
5. **副本數量**：需按被告人數提交副本+法院副本
