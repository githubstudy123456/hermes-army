# skill: financial-private-equity · diligence-checklist

> 尽职调查清单 · Due Diligence Checklist

## 基本信息

- **name**: diligence-checklist
- **description**: 系统梳理私募股权投资尽职调查各模块清单（业务、财务、法律、运营、技术），辅助分析师快速完成尽调准备工作，追踪已完成/待完成项，输出尽调完成度报告。
- **trigger**: 当用户提及"尽职调查"、"尽调"、"due diligence"、"DD"、"尽调清单"、"diligence checklist"时触发。

---

## 触发条件

- 用户输入包含 due diligence / 尽职调查 / 尽调 / DD checklist
- 项目通过初筛，需要正式进入尽调阶段
- IC 会议前需要完整尽调材料

---

## 执行步骤

### Step 1 · 确认尽调范围与时间表（web）

1. 确认基金类型（Buyout / Growth / Venture）及投资策略。
2. 确定尽调时间表（通常 4–12 周）和关键节点（IC deadline）。
3. 确认标的公司提供的数据室（Virtual Data Room）访问权限。
4. 组建尽调团队（业务/财务/法律/技术各负责人）。

### Step 2 · 业务尽调（Business Due Diligence）

1. **市场与竞争**：
   - TAM / SAM / SOM 分析
   - 客户画像（行业、地区、收入分布）
   - 客户留存率与 concentration risk
   - 竞争对手对比（产品/价格/渠道）
2. **商业模式**：
   - 收入结构（拆分各业务线贡献）
   - 单位经济模型（Unit Economics）
   - 可持续性和可规模化分析
3. **成长性**：
   - 历史增速驱动因素
   - 未来增长假设及敏感性
   - 市场渗透策略

### Step 3 · 财务尽调（Financial Due Diligence）

1. **历史财务数据**：
   - 过去 3–5 年三张表（PL / BS / CF）
   - 非经营性项目调整（Adjusted EBITDA）
   - 收入确认政策与现金流质量
2. **盈利质量**：
   - 收现比（CFS/NI）
   - 应收帐款周转天数（DSO）
   - 存货周转天数（ITO）
   - 应付帐款周转天数（DPO）
3. **估值模型支持**：
   - LBO 模型输入数据核实
   - 假设敏感性（收入增速、毛利率）
4. 使用 `terminal` 执行 Python 脚本生成财务指标汇总表。

### Step 4 · 法律尽调（Legal Due Diligence）

1. **公司治理**：
   - 股权结构（各股东持股比例、Vesting Schedule）
   - 董事会构成与决议机制
   - 关联交易情况
2. **合同合规**：
   - 重大合同审查（客户/供应商合同）
   - 知识产权（专利/商标/著作权）
   - 监管合规资质
3. **诉讼与或有负债**：
   - 正在进行的诉讼
   - 潜在诉讼风险
   - 担保/抵押情况

### Step 5 · 运营与技术尽调（Operational & Tech DD）

1. **运营体系**：
   - 核心业务流程
   - 关键供应商依赖度
   - 团队规模与结构
2. **技术尽调**：
   - 技术架构与护城河
   - 数据安全与隐私合规（GDPR/CCPA）
   - 研发管线与产品路线图

### Step 6 · 尽调清单追踪与报告输出

使用 `terminal` 生成 Markdown 尽调追踪表：

| 模块 | 尽调事项 | 负责方 | 状态 | 完成度 |
|---|---|---|---|---|
| 业务 DD | TAM 分析 | @Analyst A | 已完成 | 100% |
| 财务 DD | 历史三张表 | @Analyst B | 进行中 | 60% |
| 法律 DD | 股权结构 | @Legal | 待启动 | 0% |
| 运营 DD | 供应商依赖 | @Analyst C | 已完成 | 90% |

---

## 输出格式

- 优先输出 Markdown 表格（分模块尽调清单 + 状态追踪）
- 如需 Excel，调用 `terminal` 执行 Python 脚本（openpyxl）生成 .xlsx 清单
- 标注所有假设前提和数据来源
- 尽调完成度汇总（%）：各模块加权得出整体完成度

---

## 常见陷阱（pitfalls）

1. **尽调时间不足**：压缩尽调周期导致关键问题未深入挖掘。
2. **依赖公司提供数据**：未独立核实，假设数据准确性。
3. **忽略尾部风险**：小概率事件（监管变化、技术颠覆）未纳入分析。
4. **财务尽调不深入**：仅看 EBITDA，忽略现金流质量和资产负债表健康度。
5. **法律尽调流于形式**：未仔细审查股东协议中的保护性条款和退出条款。
