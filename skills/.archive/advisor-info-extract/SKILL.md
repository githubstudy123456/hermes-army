# 信息提炼技能

## 核心功能

从复杂信息中提炼核心，结构化输出。

## 使用场景

- 长文章/报告 → 一句话总结 + 关键点
- 会议记录 → 决议事项 + 行动项 + 待跟进
- 数据报表 → 关键数字 + 趋势判断
- 新闻事件 → 发生了什么 + 影响 + 各方反应
- 产品需求 → 核心需求 + 优先级 + 实现建议

## 输出模板

```
【信息提炼】

📌 一句话总结：
[最核心的观点，用一句话]

📊 关键数据点：
- [数据1]：说明什么
- [数据2]：说明什么

✅ 核心要点（3-5条）：
1. [要点] — [为什么重要]
2. [要点] — [为什么重要]
3. ...

⚠️ 需要注意的点：
- [可能的风险或限制]
- [容易忽略的细节]

🎯 对主人的意义：
这段信息跟主人正在做的事情有什么关系，能帮上什么忙？

❓ 值得进一步确认的问题：
1. [如果信息中有模糊的地方，列出待确认项]
```

## 信息分类处理

### 新闻/事件类
- 发生了什么（简洁）
- 谁受影响
- 各方反应
- 后续跟踪点

### 报告/文档类
- 核心结论（先行）
- 支撑结论的关键证据
- 作者立场（中性/有偏向）
- 数据新鲜度

### 聊天记录/群消息类
- 关键讨论点
- 分歧点
- 达成的共识
- 悬而未决的问题

### 数据/报表类
- 主要数字（vs 上期/同期）
- 好于/差于预期的点
- 异常波动点
- 趋势判断

## 重要原则

- 先给结论，再给细节
- 区分"事实陈述"和"我（AI）的解读"
- 如果信息不全，明确说"基于现有信息，以下是我的判断"
- 发现信息中有矛盾点，主动标注
- **每份提炼报告末尾问一句："还有其他信息我应该考虑吗？"**
