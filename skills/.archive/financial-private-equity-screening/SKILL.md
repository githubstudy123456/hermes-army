# skill: financial-private-equity · screening

> 项目筛选 · Deal Sourcing & Screening

## 基本信息

- **name**: screening
- **description**: 对潜在投资项目进行初筛，从行业、赛道、商业模式、财务指标、估值水平等维度快速评估，输出筛选评分卡与是否进入下一阶段（尽调）的建议。
- **trigger**: 当用户提及"筛选项目"、"deal sourcing"、"项目初筛"、"投资机会筛选"、"pipeline review"时触发。

---

## 触发条件

- 用户输入包含 deal sourcing / screening / 项目筛选 / 投资机会 / pipeline
- 分析师请求对目标公司进行快速初判
- PE 团队例行 pipeline review

---

## 执行步骤

### Step 1 · 收集项目基本信息（web）

1. 通过公开来源（PitchBook、Crunchbase、新闻、官网）获取标的：
   - 公司名称、成立时间、总部所在
   - 所处行业与细分赛道
   - 商业模式（产品/服务、变现方式）
   - 创始团队背景与过往经历
   - 融资历史（轮次、估值、领投方）
2. 使用 `terminal` 抓取网页或调用 API 补充信息。

### Step 2 · 行业与竞争格局分析（web）

1. 目标行业市场规模（TAM）及增速（5年CAGR）。
2. 行业竞争格局：头部玩家、市场份额、集中度。
3. 标的公司的差异化壁垒（技术/品牌/网络效应/监管资质）。
4. 政策环境：是否受监管、补贴、限制。

### Step 3 · 财务指标初筛（web + terminal）

1. 获取可获得的财务数据（营收、增速、毛利率、EBITDA）。
2. 计算关键指标：
   - 营收 CAGR（过去2-3年）
   - Gross Margin / EBITDA Margin
   - 烧钱速度（Cash Burn Rate）
   - ARR（若有订阅收入）
2. 对标行业 Comps（EV/Revenue、EV/EBITDA、P/S）。
3. 使用 `terminal` 执行 Python 脚本快速计算评分。

### Step 4 · 估值水平评估（web）

1. 最近一轮估值 vs. 行业平均估值倍数。
2. 本轮融资预期估值区间。
3. 估值是否在基金策略覆盖范围（Size Check）。

### Step 5 · 筛选评分卡输出

输出 Markdown 评分表，包含维度：

| 维度 | 权重 | 评分(1-5) | 说明 |
|---|---|---|---|
| 行业赛道 | 20% | | TAM、CAGR、格局 |
| 商业模式 | 15% | | 可规模化、壁垒 |
| 团队背景 | 15% | | 过往业绩、行业资源 |
| 财务表现 | 20% | | 增速、利润率 |
| 估值合理性 | 15% | | vs. Comps |
| 基金策略匹配 | 15% | | 规模、阶段、关注点 |

**加权总分** = Σ(评分 × 权重)，输出是否建议进入尽调。

---

## 输出格式

- 优先输出 Markdown 表格（评分卡 + 核心数据摘要）
- 如需 Excel，调用 `terminal` 执行 Python 脚本（openpyxl）生成 .xlsx 评分卡
- 标注所有假设前提（数据来源、口径）
- 输出结论：**建议进入尽调 / 暂时搁置 / 需要更多信息**

---

## 常见陷阱（pitfalls）

1. **数据来源不权威**：仅依赖新闻稿或公司官网，忽略监管文件或第三方数据库交叉验证。
2. **行业判断主观**：未量化赛道规模，仅凭"赛道热"做判断。
3. **财务数据残缺**：早期项目无公开财报，使用估算数据时未标注不确定性。
4. **估值倍数滥用**：用 ARR 倍数套用到非订阅模式公司。
5. **忽略基金策略匹配**：好项目不一定适合本基金阶段/规模/行业偏好。
