# source 目录文件列表规范

## 用户偏好（主人）
主人通过飞书访问，无法直接查看服务器文件。需要我**在聊天中直接展示**文件列表。

## 正确格式（已验证✅）

```
路径
- 文件名（大小）- 简短说明
```

### 示例

**`/home/ubuntu/`**
- `agency-agents/` - AI Agent专家目录
- `download/` - 下载文件
- `download_temp/` - 临时下载
- `physics-game/` - 物理游戏项目
- `physics-platform/` - 物理教学平台
- `source/` - 源文件目录

**`/home/ubuntu/source/`**
- `AI提示词库完整版.md`（23K）- 提示词库
- `prompts/` - 提示词子目录
- `中国土地制度历史可视化.html`（26K）- 土地制度可视化
- `中国土地制度历史重大节点.md`（6K）- 土地制度历史
- `温铁军视角-中国城乡经济分析.md`（25K）- 城乡二元结构分析
- `资金流动可视化*.html`（20-32K）- 资金流动可视化（5个版本）

## 关键要点

1. **路径单独列**，不要和文件混在一起
2. **列表用 `-`**（不是 `|` 表格）
3. **格式：`- 文件名（大小）- 说明`**
4. **不用表格**（飞书不渲染 markdown 表格）
5. **直接在聊天里列**，不要先问用户能不能看到
6. **HTML文件**：可以提供 HTTP 链接或发送文件附件

## 触发场景
- 「列一下 source 有什么」
- 「把文件发给我」
- 「看看某个目录」
