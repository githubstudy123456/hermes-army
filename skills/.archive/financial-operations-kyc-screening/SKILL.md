# KYC Screening · KYC合规筛查

## 基本信息

- **Skill Name**: financial-operations-kyc-screening
- **Description**: 客户开户文档解析与 KYC/AML 规则引擎，支持中国 AML 规定及 FATF 标准
- **Trigger Keywords**: `kyc`, `kyc screening`, `客户尽职调查`, `开户审核`, `合规筛查`
- **Category**: operations

## 核心步骤

### Step 1: 文档采集

- 通过 `file` 读取客户提交的身份证明、地址证明、资金来源证明
- 通过 `web` 查询公开制裁名单、PEP（政治敏感人士）名单

### Step 2: 文档解析

```
标准 KYC 文档清单：
1. 身份证明：身份证/护照（核验有效期 + 真实性）
2. 地址证明：水电费账单/银行月结单（3个月内）
3. 资金来源证明：收入证明/税务证明
4. 企业客户：注册文件 + 受益人信息（UBO）
```

### Step 3: AML 规则引擎（中国+FATF）

| 规则 | 标准 | 来源 |
|---|---|---|
| 制裁名单筛查 | 不在 OFAC/UN/SDN 列表 | FATF Rec 10 |
| PEP 筛查 | 非政治敏感人士 | FATF Rec 12 |
| 受益人识别 | 持股 > 25% 或实际控制人 | 中国《反洗钱法》 |
| 高风险国家 | 可疑司法管辖区 | FATF 灰/黑名单 |
| 强化尽调 | 资金来源异常/跨境频繁 | 中国《反洗钱法》 |

### Step 4: 风险评级

```
评级维度：
- 低风险：标准文档完整，无风险信号
- 中风险：文档缺失或部分无法核实
- 高风险：PEP/制裁/异常交易模式
```

### Step 5: 输出 KYC 报告

- 合规结论：通过/拒绝/需人工审核
- 风险标注
- 建议后续行动

## 工具配置

- **web**: 查询制裁名单、PEP 数据库
- **terminal**: Python OCR/文本解析 + 规则引擎
- **file**: 读取客户文档、输出 KYC 报告

## Pitfalls

1. 文档伪造（需交叉验证）
2. 壳公司伪装（UBO 穿透困难）
3. 制裁名单更新延迟
4. 文化差异导致沟通障碍

## 验证方法

- 所有文档经扫描/核验
- 规则引擎触发项均有日志
- 人工审核对高风险案例 100% 覆盖
