# 方案制定技能

## 核心功能

将模糊的目标转化为具体、可执行、有时间线的行动计划。

## 工作流程

### Step 1：确认目标和约束

明确：
- 主人想要达成什么？（目标）
- 有多少时间/资源？（约束）
- 关键里程碑是什么？

如果目标模糊，先帮主人明确化（用SMART原则）。

### Step 2：方案设计

根据目标设计 2-3 个可行方案：
- 激进方案（最快但风险高）
- 稳健方案（折中，速度和安全性平衡）
- 保守方案（最稳妥但最慢）

每个方案说明：
- 执行步骤（分阶段）
- 所需资源
- 时间线
- 预期结果
- 主要风险

### Step 3：方案推荐

推荐一个最适合的方案，给出理由。

### Step 4：落地清单

把方案变成主人可以直接执行的动作清单：

```
【行动清单】[方案名称]

第一阶段（Day 1-3）：
☐ [具体行动]
☐ [具体行动]
☐ [具体行动]

第二阶段（Day 4-7）：
☐ [具体行动]
☐ [具体行动]

第三阶段（Week 2）：
☐ [具体行动]
☐ [具体行动]

📊 进度检查点：
- [Milestone 1]：预计完成日期 X，完成标准是...
- [Milestone 2]：预计完成日期 X，完成标准是...

⚠️ 风险预案：
如果 [可能的问题] 发生，改为 [备选方案]
```

## 方案类型模板

### 项目计划
```
目标：X个月内完成Y
阶段1（需求/规划）：Z天
阶段2（执行）：Z天
阶段3（验收/调整）：Z天
```

### 学习计划
```
目标：掌握X技能
第1步：基础学习（2周）
第2步：实践应用（2周）
第3步：深化/作品集（2周）
```

### 行动实验
```
目标：验证X假设是否成立
实验设计：[简单描述]
成功标准：[可衡量的标准]
如果成功：继续X
如果失败：改为Y
```

## 重要原则

- 每个动作都是"今天就能执行"的，不是模糊的"继续努力"
- 设定检查点，不是定完计划就结束
- 如果时间/资源不足，主动说"要在X时间内完成Y，有几个地方需要取舍"
- 方案要留有余地，能应对变化
