# DPA Review · 数据处理协议审查

## 基本信息
- **Skill Name**: dpa-review
- **Description**: 审查数据处理协议（DPA），确保数据处理活动符合隐私法规
- **Trigger**: 新签DPA审查 / 现有DPA年度审计 / 供应商变更
- **Category**: privacy-legal

---

## 核心能力

### 1. DPA核心条款（GDPR Art.28 / PIPL）
| 条款类型 | 要求 |
|---------|------|
| 处理目的限制 | 明确约定处理目的，不得超范围处理 |
| 机密性义务 | 处理者须保密，不得向第三方披露 |
| 安全措施 | 技术和管理措施（加密、访问控制等） |
| 子处理授权 | 须事先取得控制者同意 |
| 数据主体权利协助 | 协助响应数据主体请求 |
| 数据删除/返还 | 目的实现后删除或返还数据 |
| 审计权 | 控制者有权审计处理者 |
| 违约责任 | 数据泄露通知义务 |
| 跨境传输 | 如涉及跨境，说明机制 |

### 2. 处理者vs控制者区分
- **控制者**：决定处理目的和方式（通常是企业自身）
- **处理者**：按控制者指示处理数据（通常是供应商/SaaS服务商）

### 3. 常见DPA风险点
| 风险类型 | 说明 |
|---------|------|
| 目的漂移 | 处理者超出约定目的使用数据 |
| 未经授权子处理 | 转包给第三方未获授权 |
| 安全措施不足 | 技术措施不符合要求 |
| 数据保留超期 | 未按约定删除数据 |
| 审计不配合 | 拒绝或拖延审计 |
| 跨境合规缺失 | 数据跨国传输无合规机制 |

---

## 操作步骤

### Step 1：读取DPA文本
- 读取数据处理协议文件
- 识别控制者和处理者身份
- 确认处理数据类型和范围

### Step 2：条款逐项审查
- 对照法规要求逐条检查
- 识别缺失或不足条款
- 评估条款有效性

### Step 3：风险评估
- 识别高风险条款
- 评估数据保护水平
- 与行业标准对比

### Step 4：修改建议
- 提供具体修改建议
- 建议增加的保护条款
- 谈判优先级排序

### Step 5：生成审查报告
- 总体评估
- 问题清单
- 建议优先级

---

## 输入要求
- DPA文本文件路径
- 处理活动描述
- 数据类型和敏感程度

## 输出
- DPA合规审查报告
- 风险条款清单
- 修改建议

---

## 注意事项（Pitfalls）
1. **处理者认定错误**：错误认定可能导致责任分配错误
2. **子处理授权缺失**：未约定子处理授权，可能违规
3. **审计条款虚设**：虽有审计条款但无操作性
4. **赔偿条款不对等**：处理者责任上限过低
5. **跨境传输机制**：如涉及跨境未说明适用机制

---

## 验证方式
- 核对核心条款完整性
- 确认风险识别准确性
- 验证修改建议可操作性
