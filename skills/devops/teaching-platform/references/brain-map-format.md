# 知识点内容文件格式说明

## 两种格式并存

teaching-platform 的 `data/{科目}/{id}.json` 存在两种格式：

### 格式A：旧格式 DICT（章节汇总）
路径示例：`物理/1-1.json`, `物理/1-2.json` 等（66个文件）

```json
{
  "id": "1-1",
  "name": "第一节 长度与时间的测量",
  "star": 2,
  "引入": { "类比说明": "...", "生活场景": [...] },
  "知识点": [
    {
      "title": "一、长度的单位",
      "讲解": "长度的基本单位是米（m）...",
      "关键公式": "1 km = 1000 m",
      "生活实例": ["课桌高度约75 cm"],
      "典型模型": [{ "name": "...", "desc": "...", "scene": "..." }]
    }
  ],
  "小结": { "要点": [...], "记忆口诀": "..." },
  "练习": [{ "type": "choice", "question": "...", "options": [...], "answer": 0, "analysis": "..." }]
}
```

**问题**：内容大量是 placeholder（模板填充），典型模型张冠李戴（力学内容塞进声学章节）。

---

## 如何区分两种格式

```python
import json
with open(path) as f:
    data = json.load(f)
if isinstance(data, list):
    print("LIST/brain-map 格式（当前标准）")
elif isinstance(data, dict):
    print("DICT/旧格式（待重写）")
```

## brain-map 格式写作规范

1. **content**：200-500字，面向零基础学生，要有"为什么"和"生活实例"
2. **summary**：30-60字，一句话概括，便于脑图节点展示
3. **modelKind**：`"none"` — 一般知识点；`"3d"` — 需要3D模型演示（如电磁铁、凸透镜成像、滑轮组）
4. **id**：严格 `{章节号}-{知识点序号}`，如 `1-1-1` 表示第1章第1节第1个知识点
5. **每节知识点数**：通常 3-5 个，不要拆分太细，也不要合并太多

## 前端渲染逻辑

brain-map 格式文件被 `loadSection()` 读取后，前端渲染为脑图节点列表。
点击节点展示 `content` 详情。星图展示 `summary`。
