## 实体合规审查

### 触发条件
- 用户上传营业执照、企业年报或合规自查文件
- 用户询问"公司合规"、"工商年检"、"税务登记状态"等
- 投资尽调或定期合规审查场景

### 执行步骤
1. **接收企业基本信息**：公司类型、注册资本、成立日期、经营范围
2. **核查营业执照登记事项**：与实际运营状况一致性
3. **年检与公示信息核查**：国家企业信用信息公示系统年报提交情况
4. **税务合规状态**：纳税人资格、报税记录、税务稽查情况
5. **社保及公积金缴纳核查**：全员参保情况及基数合规性
6. **外汇登记核查**：外商投资企业外汇登记证（FIE）及资本金结汇
7. **行业专项许可**：如食品经营许可证、ICP许可证等
8. **输出合规评估报告**：分模块标注合规/待整改/缺失项，并提供整改建议

### Pitfalls
- 仅核查中央法规，忽视地方工商部门的具体操作要求
- 将"已完成年检"等同于"合规运营"，忽视年报内容真实性
- 对外商投资企业遗漏商务部门审批备案要求
- 忽视经营范围变更未及时更新登记的法律风险

### Verification
- 报告覆盖工商、税务、社保、外汇四大基础模块及行业专项（如适用）
- 每项核查结果标注法规依据（如《公司法》第X条）
- 待整改项标注预计完成时间及责任部门
- 可通过 web tool 查询国家企业信用信息公示系统验证公示信息

## 董事会会议记录

### 触发条件
- 用户上传董事会/股东会会议记录文本或文档
- 用户要求生成符合法律规范的会议决义摘要
- 用户查询"董事会决议"、"会议通过"等关键词

### 执行步骤
1. **接收原始记录**：读取用户提供的会议记录文件或文本
2. **识别会议类型**：判断是股东会、董事会还是专业委员会会议
3. **提取关键要素**：时间、地点、出席人员、议程、表决结果、签字
4. **生成结构化摘要**：按法律规范格式输出决议事项、表决情况、生效条件
5. **核查合规性**：对照《公司法》第37/46/67条及公司章程核对完整性

### Pitfalls
- 遗漏法定出席人数及表决比例记录
- 未区分普通决议与特别决议的表决要求
- 缺少会议召集程序（召集人、通知时限）的说明
- 决议文本与会议记录时间不一致

### Verification
- 输出的会议记录包含：会议时间、地点、主持人、记录人、出席人员、缺席人员及原因、议程、每项议题的讨论要点、表决结果（同意/反对/弃权）、签字
- 特别决议事项已标注《公司法》第44条的2/3表决权要求
- 可通过 file tool 读取历史记录验证格式一致性

## 尽调问题提取

### 触发条件
- 用户上传尽调报告或访谈记录文本
- 用户询问"尽调问题"、"风险点"、"合规瑕疵"等
- 项目立项或投资决策阶段需要梳理风险清单

### 执行步骤
1. **接收尽调材料**：读取尽调报告、访谈纪要或问题清单文件
2. **识别问题类别**：股权结构、知识产权、劳动人事、诉讼仲裁、环保合规、税务等
3. **提取问题要点**：问题描述、涉及主体、涉及金额（如有）、发现来源
4. **进行风险定级**：Red（重大障碍）/ Amber（需关注）/ Green（轻微或无实质影响）
5. **建议处理方案**：披露/整改/赔偿承诺/放弃交易等
6. **输出结构化问题清单**：按优先级排序，含问题描述、风险等级、建议措施

### Pitfalls
- 将法律与商业问题混淆，未区分纯法律风险
- 对 red-flag 问题未深入核查即建议"带病推进"
- 忽视地方性法规与中央法规的差异（如环保地方标准）
- 问题定级缺乏量化依据（如涉及金额、影响人数）

### Verification
- 每项问题均有风险等级标注及定级理由说明
- 涉及金额或人数的问题已量化呈现
- 同类问题已归类汇总，避免重复列举
- 可通过 file tool 读取历史尽调报告验证分级标准一致性

## 交易交割清单

### 触发条件
- 用户上传交易协议文本或请求生成交割清单
- 用户询问"交割条件"、"先决条件"、"closing checklist"等
- 交易执行阶段需要逐项核查进展

### 执行步骤
1. **识别交易类型**：股权收购、资产收购、增资扩股、合并等
2. **生成交割先决条件清单**：基于交易协议第X条列出所有先决条件（Conditions Precedent）
3. **核查每项条件状态**：满足/待满足/豁免，附说明及负责方
4. **文件签署完整性检查**：对照签署页确认各方签字盖章
5. **资金与股权交割验证**：确认对价支付路径、股权登记完成
6. **生成交割确认书模板**：汇总各项条件达成情况，各方签署

### Pitfalls
- 将监管审批（重组委、商务部）误列为"可豁免"条件
- 未区分"交割前必须满足"与"交割后持续义务"
- 忽视少数股权优先购买权等股东保护条款
- 对境外交易遗漏外汇登记核查项

### Verification
- 先决条件清单每项标注状态（Met/Unmet/Waived）及核查日期
- 资金交割路径与协议约定一致（托管账户/直接支付）
- 交割后工商变更登记在约定期限内完成
- 可通过 file tool 比对协议原文与清单对应条款

## 重大合同台账

### 触发条件
- 用户上传合同清单或企业ERP导出的合同台账
- 用户询问"重大合同"、"合同到期"、"关联交易合同"等
- 年度法务报告或审计配合场景

### 执行步骤
1. **接收合同清单**：读取用户提供的Excel/Word合同台账或合同文本
2. **识别重大合同标准**：单笔金额≥1000万、关联交易所涉、担保/抵押合同、境外合同
3. **提取合同关键信息**：合同编号、签约方、合同类型、签订日期、到期日期、金额、主要权利义务
4. **分类汇总**：按合同类型（借款担保/采购销售/知识产权/关联交易）分类统计
5. **到期预警分析**：60天内到期合同标注预警，按紧急程度排序
6. **履约状态追踪**：正常履行/违约/纠纷/已终止分类标注
7. **生成台账报告**：结构化呈现合同清单及统计分析

### Pitfalls
- 未区分"重大合同"与"一般合同"，导致台账过于冗长
- 将已终止合同与在履行合同混同统计
- 遗漏合同变更/补充协议的追踪
- 关联交易未按监管要求单独列示并披露金额

### Verification
- 合同台账包含：合同编号、签约方、类型、金额、签订日、到期日、履约状态
- 到期预警（60天内）已单独标注并按剩余天数排序
- 关联交易台账与上市公司披露口径一致，可比对年报
- 可通过 file tool 读取历史台账验证数据连续性
