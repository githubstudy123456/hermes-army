# Launch Watcher · 产品上市合规审查

## 基本信息
- **Skill Name**: launch-watcher
- **Description**: 产品上线前合规checklist审查，确保符合中国网络法规要求
- **Trigger**: 产品开发完成 / 上线前30天 / 重大功能变更
- **Category**: product-legal

---

## 核心能力

### 1. 中国网络法规体系
| 法规 | 生效时间 | 核心要求 |
|------|---------|---------|
| 网络安全法 | 2017.6.1 | 网络安全等级保护、数据本地化、关键基础设施保护 |
| 数据安全法 | 2021.9.1 | 数据分类分级、重要数据保护、跨境数据管理 |
| 个人信息保护法 | 2021.11.1 | 个人信息处理规则、敏感信息保护、跨境传输 |
| 电子商务法 | 2019.1.1 | 电商平台责任、消费者保护、知识产权 |
| 互联网广告法 | 2016.9.1 | 广告标识、禁止虚假宣传、未成年人保护 |

### 2. 产品上线前合规checklist

#### 基础合规项
- [ ] 隐私政策发布且可访问
- [ ] 用户协议（ToS）发布且可访问
- [ ] Cookie政策（如适用）
- [ ] 用户账户注销功能
- [ ] 个人信息查阅/更正功能
- [ ] 未成年人保护措施（如面向未成年用户）
- [ ] 知识产权权属清晰（商标/专利/著作权）

#### 网络安全法合规
- [ ] 完成网络安全等级保护备案（如需）
- [ ] 制定网络安全事件应急预案
- [ ] 建立日志留存机制（不少于6个月）
- [ ] 采取防范网络攻击措施

#### 数据安全法合规
- [ ] 完成数据分类分级
- [ ] 制定重要数据保护措施
- [ ] 建立数据安全负责人
- [ ] 制定数据安全培训计划

#### PIPL合规
- [ ] 收集个人信息获得单独同意
- [ ] 敏感信息处理获得单独同意
- [ ] 建立个人信息主体权利响应机制
- [ ] 制定跨境传输方案（如涉及）
- [ ] 完成个人信息保护影响评估（如需）

#### 电商/内容合规（如适用）
- [ ] 平台规则公示
- [ ] 知识产权侵权投诉机制
- [ ] 用户生成内容审核机制
- [ ] 广告标识合规

### 3. 分级评估
| 风险等级 | 评估标准 | 上线前需确认 |
|---------|---------|------------|
| 高 | 处理敏感信息/面向未成年人/金融医疗 | 完整法律评估+律师签字 |
| 中 | 处理一般个人信息/用户量较大 | 基础合规checklist |
| 低 | 工具类/无个人信息处理 | 快速检查 |

## 操作步骤

### Step 1：产品信息收集
- 读取产品说明文档
- 收集：产品类型、用户群体、数据类型、功能列表
- 确认是否涉及特殊行业（金融、医疗、教育等）

### Step 2：适用法规识别
- 识别适用的法规要求
- 区分基础合规和行业特定要求
- 评估风险等级

### Step 3：合规checklist执行
- 逐项检查checklist
- 收集佐证材料
- 识别缺口

### Step 4：差距分析和整改建议
- 输出差距分析报告
- 给出优先级排序
- 提供整改方案

### Step 5：生成上线前合规意见书
- 总体合规评估
- 遗留问题及处理计划
- 上线建议

---

## 输入要求
- 产品说明文档路径
- 功能清单
- 数据处理活动描述
- 目标上线日期

## 输出
- 产品上线前合规checklist
- 差距分析报告
- 整改建议
- 合规意见书草稿

---

## 注意事项（Pitfalls）
1. **行业许可证缺失**：特定行业（医疗、金融）需预先取得许可证
2. **备案延误**：ICP备案、等级保护备案需提前办理
3. **第三方SDK合规**：集成SDK需确保其合规性
4. **跨境场景忽视**：如用户含海外用户，需额外考虑GDPR等
5. **未成年人保护**：需单独评估和披露

---

## 验证方式
- 核对checklist完整执行
- 确认高风险项已专项审查
- 验证合规意见书逻辑清晰
