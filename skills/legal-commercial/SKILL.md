## NDA 快速分类

### 触发条件
- 当主人上传 NDA 文件或粘贴 NDA 文本并要求"审一下"、"分类"、"风险"时自动激活

### 三级分类标准

**GREEN — 低风险（标准条款，可直接签署）**
| 判断条件（满足任意一条） |
|------------------------|
| 双向 NDA，且保密期限 ≤ 2 年 |
| 保密信息定义合理，无过度扩展 |
| 无单方面修改权 |
| 无不合理竞业/禁止招揽条款 |
| 违约救济限于实际损失，不含惩罚性赔偿 |
| 适用法律/管辖法院为中国大陆 |

**YELLOW — 中等风险（需关注，部分条款建议谈判）**
| 判断条件（满足任意一条） |
|------------------------|
| 单向 NDA（一方仅承担义务，无对等权利） |
| 保密期限 > 3 年 |
| 保密信息定义过宽（包含"一切信息"等表述） |
| 存在自动续约条款（无明确终止通知机制） |
| 违约救济含惩罚性赔偿（超出实际损失） |
| 适用法律/管辖法院为境外 |
| 包含 IP 归属约定但不明确 |

**RED — 高风险（重大风险，建议拒绝或重大修改）**
| 判断条件（满足任意一条） |
|------------------------|
| 无保密期限限制（永久有效） |
| 保密信息定义极度宽泛（含"合作本身"等） |
| 包含禁止招揽条款且限制期 > 2 年 |
| 单方面赋予对方修改协议权利 |
| 违约救济含三倍赔偿或律师费转移 |
| 适用法律为对中国当事人不利的境外法域 |
| 强制要求转让 IP 所有权 |

### 工作流程
```
Step 1：读取合同 → Step 2：识别 NDA 类型 → Step 3：扫描 RED 指标
→ Step 4：扫描 YELLOW 指标 → Step 5：无 RED/YELLOW → GREEN → Step 6：输出报告
```

## 供应商合同审查

### 触发条件
- 当主人上传供应商合同文件或粘贴文本并要求"审合同"、"供应商合同审查"、"看看有没有坑"时自动激活

### 重点审查条款
1. **合同标的与范围**：服务/商品描述、交付物定义、变更机制
2. **价格与付款**：定价机制、涨价限制、付款周期、隐性费用、滞纳金
3. **质量保证**：质保期限、质保范围、验收标准
4. **知识产权**：IP 归属、授权范围、背景 IP
5. **保密义务**：保密期限、保密范围、保密例外
6. **违约责任**：赔偿上限、间接损失、律师费、liquidation damages
7. **合同终止**：终止权、终止通知期、自动续约陷阱、终止后义务
8. **适用法律与争议解决**：适用法律、争议解决机制

### 风险评分标准（1-10）
| 分数区间 | 风险等级 | 含义 |
|---------|---------|------|
| 1-3 | 🟢 低风险 | 标准条款，对我方基本无害 |
| 4-6 | 🟡 中等风险 | 需要关注，可能需要谈判 |
| 7-10 | 🔴 高风险 | 需要修改，可能造成实质损失 |

## SaaS MSA 审查

### 触发条件
- 当主人上传 SaaS 服务协议（MSA+SOW）并要求"MSA审查"、"服务协议"、"看合同条款"时自动激活

### 重点审查模块
1. **数据处理条款（DPA）**：数据所有权、数据跨境、违约数据处理
2. **可用性条款（SLA）**： uptime 承诺、补偿机制、维护通知
3. **赔偿条款（Indemnification）**：IP 侵权赔偿、责任上限
4. **IP 归属**：我方数据/内容的 IP 归属、模型训练数据
5. **自动续约与定价**：自动续约条款、价格变动限制

### 风险评分标准（1-10）
同供应商合同审查。

## 合同续约追踪

### 触发条件
- 当主人要求"查一下哪些合同要到期了"、"续约追踪"、"合同到期提醒"、"最近有哪些合同要续约"时自动激活

### 预警分级
| 预警等级 | 触发时间 | 说明 | 行动 |
|---------|---------|------|------|
| 🔴 紧急 | 到期前 ≤ 30 天 | 需立即处理 | 立即生成续约决策报告 |
| 🟡 预警 | 到期前 31-60 天 | 需关注 | 生成续约决策报告，开始准备 |
| 🟢 观察 | 到期前 61-90 天 | 提前规划 | 生成续约摘要，列入监控 |

### 续约选项分析
| 选项 | 说明 | 风险 | 建议 |
|------|------|------|------|
| A. 续约（原条件） | 按原合同条件续约 | 低 | 如果原条件有利 |
| B. 续约（重新谈判） | 续约但重新谈判条款 | 中 | 如果需要更新条款 |
| C. 不续约（终止） | 合同到期终止，不续约 | — | 如业务不再需要 |
| D. 更换供应商 | 终止后更换为其他供应商 | 高 | 如现有供应商不满意 |
| E. 转为永久/长期 | 短期合同转为长期合同 | 中 | 如长期需求明确 |

### 台账字段说明
| 字段 | 类型 | 说明 |
|------|------|------|
| contract_id | string | 合同编号 |
| contract_name | string | 合同名称 |
| counterparty | string | 对方当事人 |
| contract_type | enum | NDA/供应商/SaaS/采购/其他 |
| amount | decimal | 合同金额（万元） |
| end_date | date | 到期日期 |
| auto_renewal | boolean | 是否自动续约 |
| notice_period | int | 提前通知天数 |
| status | enum | active/pending_renewal/terminated |

## 风险升级路由

### 触发条件
- 当主人询问"这个合同要不要升级"、"风险大不大"、"是否需要法务介入"时自动激活

### 升级判断矩阵
| 因素 | 非升级（直接处理） | 升级（法务介入） |
|------|-----------------|----------------|
| 合同金额 | < 50万 | ≥ 50万 |
| 风险评分 | 1-3（低风险） | 4-10（中/高风险） |
| 合同类型 | 标准模板 | 非标准模板/定制条款 |
| 对方当事人 | 无争议历史 | 有诉讼/违约历史 |
| 涉及事项 | 常规业务 | IP/数据/竞业等高敏感事项 |

### 升级路径
| 风险等级 | 升级对象 | 响应时间 |
|---------|---------|---------|
| 🔴 RED + 高金额 | 首席法务官 | 24小时内 |
| 🔴 RED 或 高金额(≥50万) | 法务总监 | 48小时内 |
| 🟡 YELLOW + 中金额(≥20万) | 法务经理 | 5个工作日内 |
| 🟢 GREEN | 自主处理 | — |

## 通用输出格式

所有模块输出必须包含：
- 标注"草稿，需律师审核确认"
- 引用条款标注 [verify] 需人工核实
- consequential actions（发送、归档、执行）需用户确认

---

## 通用 Pitfalls

- 引用来源标注 [verify]：发现风险条款时标注，需人工核实
- 输出标注：所有输出必须注明"草稿，需律师审核确认"
- 不承诺结果：不保证审查绝对准确
- 律师-委托人特权：发现 privileged 内容时注意保护
