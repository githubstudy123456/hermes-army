# 每周生活选题推送 — 输出格式规范

**来源：** 2026-06-16 主人反馈格式正确，要求保持

---

## 标准格式（单条 + 加粗）

AI 生成内容后，**每条内容单独发一条消息**，格式如下：

```
**① 标题文字（加粗）**
来源平台名 · 日期
📍 地点标签

内容简介，150字以内，介绍这个体验是什么、为什么有趣。

为什么有趣：一句话说明
适合形式：短视频/图文/直播/文章
```

每条之间空一行分隔。

**示例：**

```
**① 在巴厘岛数字游民聚居地工作一个月是什么体验**
National Geographic Travel · 2026-06-10
📍 巴厘岛·仓古

深入探访仓古村数字游民社区，记录来自欧美的远程工作者如何用平均每月1200美元在巴厘岛过上包住宿、瑜伽、冲浪的慢生活。探讨数字游民文化对当地房租和社区关系的改变。

为什么有趣：真实的第一手生活成本数据 + 东西方工作文化碰撞
适合形式：图文长文
```

---

## 关键规则

1. **每条单独发送**，不合并成一条消息（避免飞书渲染后格式混乱）
2. **标题必须加粗**（`**标题**`）
3. **用①②③④序号**，不用 `1.` `2.`
4. 每条之间空一行
5. 目标群：`feishu:oc_605cb68f2814b7fef2336ae15b7982bd`（周报群）

## cron run 的真实行为（踩坑记录）

**`cron run` 只是重新排期，不是立即执行。**

```
$ cronjob run job_id=6b1782f0b52b
→ 返回 success: true，next_run_at 更新为"立即"
→ 但内容并没有推送
```

原因：cron run 把 `next_run_at` 更新为当前时间，但实际执行依赖于 cron 调度器的下一次 tick。

**立即测试的正确方式：**
1. 直接用 `send_message` 发测试内容到目标群
2. 或者等待下一个调度周期（约1-2分钟内）

**不要误以为 run 成功=推送成功。** 验证是否真正执行：等几秒后去飞书群看有没有新消息。
