## AML 核查

### 触发条件
- 当主人要求"AML核查"、"反洗钱"、"洗钱筛查"时自动激活
- 新客户开户、持续监控、高风险客户定期审查场景

### 执行步骤
**Step 1 · 收集客户信息**
- 客户基本信息：名称、注册地址、实际控制人、股权结构
- 交易信息：预期交易规模、交易对手、交易地区
- 行业信息：所在行业、商业模式、预期资金流动特征

**Step 2 · 名单筛查**
- 通过 `web` 或公开数据库筛查：
  - 制裁名单（OFAC、EU、UN、中国制裁名单）
  - 政治敏感人士（PEP）
  - 反恐融资名单
  - 可疑交易对手数据库
- 使用 `terminal` 执行 Python 脚本进行名单比对

**Step 3 · 风险评估**
| 风险因素 | 高风险信号 | 低风险信号 |
|---------|-----------|-----------|
| 客户类型 | 离岸公司、空壳公司 | 上市公司、监管持牌机构 |
| 交易对手 | 高风险司法管辖区 | 同等司法管辖区 |
| 交易模式 | 大量现金、快速资金转移 | 常规银行转账 |
| 行业 | 博彩、珠宝、虚拟资产 | 零售、制造、科技 |
| PEP 关联 | 是 | 否 |

**Step 4 · 可疑交易识别**
- 结构化交易（Smurfing）：大量小额分散交易
- 跨境大额转账：高风险司法管辖区
- 关联方交易：缺乏合理商业目的
- 频繁更换交易对手：清洗资金痕迹

**Step 5 · 生成 AML 报告**
```
# AML 核查报告

客户名称：[名称]
核查日期：[日期]
核查类型：[新客户开户/持续监控/定期审查]

【名单筛查结果】
- 制裁名单：✅ 未命中 / ⚠️ 命中（[名单名称]）
- PEP：✅ 否 / ⚠️ 是（[关系]）
- 可疑对手：✅ 未发现 / ⚠️ 发现（[对手名称]）

【风险评估】
- 风险等级：[🟢低风险/🟡中等风险/🔴高风险]
- 风险因素：[列出主要风险因素]

【可疑交易识别】
- 结构化交易：✅ 未发现 / ⚠️ 发现
- 跨境大额转账：✅ 未发现 / ⚠️ 发现
- ...

【建议行动】
- [🟢 正常开户 / 🟡 强化尽调 / 🔴 拒绝开户 / 提交可疑交易报告]

备注：本报告为草稿，需合规部门确认，不构成合规结论。
```

### 常用监管依据
- FATF 40 项建议
- 中国人民银行反洗钱规定
- FATF 互评估报告（国家风险）
- OFAC 制裁法规

### Pitfalls
1. 名单数据过时：需定期更新制裁名单数据库
2. PEP 识别不完整：需覆盖所有关联人（实控人、高管、家庭成员）
3. 忽视行业风险：特定行业（虚拟资产、现金密集型）需强化审查
4. 交易监控阈值不合理：阈值过低导致过多误报

### Verification
- [ ] 名单筛查覆盖所有适用名单
- [ ] PEP 关联人识别完整
- [ ] 风险评估有量化依据
- [ ] 可疑交易识别有具体交易记录支撑

## KYC 筛查

### 触发条件
- 当主人要求"KYC筛查"、"客户尽职调查"、"客户身份核验"时自动激活
- 客户开户、持续尽调、供应商审查场景

### 执行步骤
**Step 1 · 收集身份信息**
- 企业客户：公司注册文件、营业执照、股权结构图、实际控制人身份证明
- 自然人客户：身份证明、地址证明、职业信息
- 通过 `web` 核查公开信息（注册信息、司法记录、新闻）

**Step 2 · 身份核验**
1. **注册信息核实**：公司注册号、注册地、存续状态
2. **实控人识别**：穿透至最终受益人（UBO，持股≥10%或实际控制）
3. **身份证明核实**：证件真实性、有效期
4. **地址证明核实**：真实性检验

**Step 3 · 受益人识别（UBO）**
| 持股比例 | 是否为 UBO |
|---------|-----------|
| ≥25% | ✅ 是 |
| 10-25% | 需结合控制权判断 |
| <10% | 通常否，除非有其他控制权证据 |

**Step 4 · 高风险识别**
- 客户位于高风险司法管辖区（FATF 高风险国家）
- 客户为 PEP（政治敏感人士）或与 PEP 关联
- 客户从事高风险行业（博彩、虚拟资产、现金密集型）
- 股权结构不透明（复杂嵌套结构、空壳公司）

**Step 5 · 生成 KYC 报告**
```
# KYC 筛查报告

客户名称：[名称]
客户类型：[企业/自然人]
核查日期：[日期]

【身份核验】
- 注册信息：✅ 核实 / ❌ 存疑（[说明]）
- 实际控制人：✅ 已识别 / ❌ 未识别（[说明]）
- 身份证明：✅ 有效 / ❌ 无效或过期

【UBO 识别】
- UBO 姓名：[姓名]
- 持股比例：X%
- 身份证明：✅ 已核验 / ❌ 待核验

【风险评估】
- 风险等级：[🟢低风险/🟡中等风险/🔴高风险]
- 高风险因素：[列出]

【建议行动】
- [🟢 正常开户 / 🟡 强化尽调（补充材料）/ 🔴 拒绝开户]

备注：本报告为草稿，需合规部门确认，不构成合规结论。
```

### 常用监管依据
- 中国人民银行《金融机构客户身份识别和客户身份资料及交易记录保存管理办法》
- FATF 建议（建议10、12-21）
- 中国银保监会《商业银行合规管理指引》

### Pitfalls
1. UBO 穿透不完整：未穿透至最终受益人
2. 身份证明伪造：未通过多渠道交叉验证
3. 高风险行业识别遗漏：未覆盖特定高风险行业
4. 文件过期：使用过期的注册文件或身份证明

### Verification
- [ ] UBO 穿透至最终受益人
- [ ] 身份证明通过多渠道核验
- [ ] 高风险因素已完整识别
- [ ] 文件有效期在核查日期内
