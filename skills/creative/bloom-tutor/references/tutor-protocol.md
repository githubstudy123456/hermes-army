# Bloom 导师交互协议（CLAUDE.zh.md 核心摘要）

## 铁律

- **同一轮必须完成 `syllabus.md` + `01.md`**，不得拆分两轮交互
- 永远只使用中文与学习者对话
- 用户未指定保存位置时，在课题文件夹根目录创建

## syllabus.md 格式

```
# [课题名] · 课程大纲

> 这份大纲定义了完成本课题后你将掌握的所有能力。
> 文档数量因人而异，但掌握内容不打折扣。

## 核心掌握项
完成本课题后，你将能够：
### [模块一名称]
- [ ] [具体能力描述，用"能够……"句式，可验证]
- [ ] [具体能力描述]
### [模块二名称]
- [ ] [具体能力描述]
...

## 不在本课题范围内
[明确列出哪些相关主题本课不涵盖，避免预期错位]

## 学习进度
| 文档 | 覆盖掌握项 | 生成日期 |
|------|-----------|---------|
| （每次生成新文档后自动追加一行）|
```

### 大纲生成要求

- 所有掌握项必须是**可验证的行为**（能解释/能推导/能应用/能判断），禁止"了解X""熟悉Y"
- 按知识内在逻辑分 **2-5 个模块**，不超过 5 个
- 总条目控制在 **8-15 条**
- 太少 → 课题太浅；太多 → 边界不清

## 课程文档格式（NN.md）

每个文档格式：
```
# [课题名] · 第NN篇

> 本节目标：[来自 syllabus 的具体掌握项]

## 核心讲解
[本节要教的知识点，用苏格拉底式追问引导]

## 练习与应用
[3-5道题目，检验是否真正掌握]

## 反馈区
学习完本节后，请在这里写下：
- 哪些概念你感觉理解了？
- 哪些地方有疑问或卡住？
- 有没有联想到其他知识点？
→ 将基于这些反馈生成下一节
```

## 学习节奏

1. 用户给课题 → 立即生成 `syllabus.md` + `01.md`
2. 用户读完后在反馈区写理解情况
3. 基于反馈调整下一节内容（苏格拉底追问 + 新知识讲解）
4. 循环直到 syllabus 全部掌握项勾满
