## 轮换规则

### 第一阶段：第一批（12家）轮换

1. **每次执行前先扫描历史文件** `~/.hermes/cron/output/daily-research-*.md`，统计第一批中每家公司已被调研过的次数
2. **已调研过的公司全部跳过** — 每家在整个池子轮完之前不得重复入选
3. 第一批12家全部调研过一轮后，进入第二阶段

### 第二阶段：第二批（全球大公司）轮换

1. 从第二批全球大公司池中按顺序选取，已调研过的跳过
2. 第二批全部轮完后，重置追踪表，再从第一批未调研的公司继续（此时第一批已全部调研过，逻辑上从第二批重新开始即可）

**比亚迪优先**：只要还未调研过，不论哪个阶段轮到都选比亚迪。

**今日已调研过的公司不得重选** — 即使行业相同也要换下一家。

**今日教训（2026-06-16 阿里调研）：**
- 尝试了新浪/腾讯/东方财富/Yahoo/StockAnalysis等14+个数据源全部失效或被拦截
- 最终可用：NASDAQ API（BABA价格$112.82）、Bing中文搜索（新闻）、Bing英文搜索（新闻）
- 财务数据：只能通过并行搜索新闻片段+已知公开信息拼凑，无法获取原始三张报表
- **结论：遇到API大规模失效时，立即转向Bing搜索+BrowserNavigate，不要在无效API上重复尝试**

2. **Bing中文搜索** → `https://cn.bing.com/search?q={关键词}` — 中文新闻搜索成功率最高
3. **Bing英文搜索** → `https://www.bing.com/search?q={关键词}` — 英文财经新闻
4. **`delegate_task` 并行web搜索** → 获取新闻和研报数据（需避免直接用curl pipe to python）
5. **`browser_navigate` 浏览IR页面** → 需处理Cookie Consent弹窗（点击"允许所有"），获取财报原文最可靠
6. **明确标注数据暂缺** — 当所有渠道都无法获取财务数据时，在报告中如实标注数据来源和估算范围

## 财务数据采集降级策略（重要）

当公司财报数据无法通过API或搜索获取时，**禁止捏造数据**。降级路径：

1. **明确标注"数据暂缺，待官方公告"** — 在财务概况章节如实说明
2. **尝试更多搜索关键词**：
   - `"{公司名}" "业绩" "2024" site:hkex.com.hk`
   - `"{股票代码}.HK 业绩"` 或 `"{股票代码} 年报"`
   - 公司名 + "20-F" 或 "6-K"（美股上市公司SEC文件）
   - 中文公司 + "年报" "营业收入" "净利润"
3. **财务数据估算校验**：如果不得不估算，必须标注"估算"且给出合理范围（如"亏损约9-11亿美元"），禁止写单一具体数字
4. **PE异常值处理**：港股PE字段（index 43）可能因亏损或非经常性损益显示异常值（如3.04对BeiGene明显偏低），**不要盲目引用**，在报告中说明"PE参考性有限"
5. **市场资本校验**：获取市值数据后，用 `总市值 / 股价 ≈ 股本` 快速校验是否在合理范围（如BeiGene股价HK$183×股本约8.4亿股≈市值HK$1537亿，与API返回值量级一致）

新闻条目**必须有具体来源和时间**，禁止写无来源的模糊事件：

## 路径注意事项

`~/.hermes/cron/output/` 目录在某些部署下属主是 `root`，权限 `drwx------`，不可直接写入。

**正确路径**（cron 实际运行时使用）：
```
/home/ubuntu/.hermes/cron/output/daily-research-{date}.md
```

cron 调用时，shell 环境 `$HOME` 可能为 `/root`，但 cron 实际以 `ubuntu` 用户运行，因此：
- 不要依赖 `~/.hermes/...` 自动解析
- 始终使用绝对路径 `/home/ubuntu/.hermes/cron/output/`
- 写入后用 `read_file` 读取内容作为最终回复输出
